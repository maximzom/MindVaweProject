## 🎯 Про проєкт

**MindWave** — це інтелектуальний веб-ресурс, розроблений для допомоги користувачам у побудові збалансованого розкладу дня. Проєкт поєднує сучасний веб-дизайн з AI-асистентом на базі Google Gemini, який надає персоналізовані рекомендації щодо:

- 🎯 **Фокусу та концентрації** — сесії глибокої роботи по 45-60 хвилин
- ☕ **Легкої роботи** — рутинні задачі без перевантаження
- 🌙 **Вечірнього релаксу** — плавний перехід від роботи до відпочинку
- 😴 **Підготовки до сну** — ритуали для якісного сну

### 💡 Концепція

MindWave базується на науково обґрунтованій методології **"Фокус → Пауза → Відновлення"**, яка допомагає:
- Підтримувати продуктивність протягом дня
- Уникати перевтоми та вигорання
- Формувати здорові цифрові звички
- Покращити якість сну

## ⚙️ Налаштування

### 1. Отримання API ключа Gemini

1. Перейдіть на [Google AI Studio](https://aistudio.google.com/app/apikey)
2. Увійдіть у свій Google акаунт
3. Натисніть **"Create API Key"**
4. Скопіюйте згенерований ключ

### 2. Налаштування API ключа

Відкрийте файл `chatbot.js` та знайдіть рядок:

```javascript
const MW_GEMINI_API_KEY = "YOUR_API_KEY_HERE";
```

Замініть `YOUR_API_KEY_HERE` на ваш реальний ключ:

```javascript
const MW_GEMINI_API_KEY = "AIzaSyBOTVVC5GiQfQdJj97t2Z5NbKIiyrfTItc";
```

> ⚠️ **ВАЖЛИВО**: Ніколи не публікуйте свій API ключ у публічних репозиторіях! Додайте `chatbot.js` до `.gitignore` або використовуйте змінні середовища.

### 3. Налаштування моделі (опціонально)

За замовчуванням використовується `gemini-2.0-flash-exp`. Щоб змінити модель:

```javascript
const MW_GEMINI_MODEL = "gemini-2.0-flash-exp"; // Швидка модель
// або
const MW_GEMINI_MODEL = "gemini-pro";           // Більш потужна модель
```

### 4. Налаштування системного промпта

У функції `mwAskGemini()` ви можете змінити поведінку бота:

```javascript
const systemInstruction = {
  parts: [{
    text: "Ти — асистент MindWave. Допомагай з режимом дня. Відповідай українською. Використовуй списки."
  }]
};
```

## 🤖 Функціонал чат-бота

### Основні можливості

#### 1. Підтримка контексту діалогу
```javascript
// Приклад збереження контексту
session.history = [
  { role: "user", text: "Налаштуй фокус" },
  { role: "bot", text: "Звичайно! Ось кроки..." },
  { role: "user", text: "А що робити з телефоном?" },
  { role: "bot", text: "Щодо телефону з попереднього фокусу..." }
];
```

#### 2. Категоризація запитів
Бот автоматично визначає тему звернення:
- 🎯 **Фокус** — слова: "фокус", "концентрація"
- 😴 **Сон** — слова: "сон", "заснути"
- ☕ **Перерва** — слова: "перерва", "пауза"
- 😰 **Тривожність** — слова: "тривога", "хвилювання"
- 🔥 **Перевтома** — слова: "вигорання", "перевтома"

#### 3. Статистика взаємодії
```javascript
{
  totalMessages: 45,
  userMessages: 23,
  botMessages: 22,
  categories: {
    "Фокус": 8,
    "Сон": 5,
    "Перерва": 3,
    "Тривожність": 4,
    "Перевтома": 3
  }
}
```

### API Integration

#### Структура запиту до Gemini

```javascript
{
  system_instruction: {
    parts: [{
      text: "Системний промпт..."
    }]
  },
  contents: [
    {
      role: "user",
      parts: [{ text: "Перше повідомлення користувача" }]
    },
    {
      role: "model",
      parts: [{ text: "Відповідь бота" }]
    },
    {
      role: "user",
      parts: [{ text: "Друге повідомлення користувача" }]
    }
  ]
}
```

#### Обробка відповіді

```javascript
const data = await response.json();
const botReply = data.candidates[0].content.parts[0].text;
```

   ↓

## 🐛 Відомі проблеми та рішення

### Проблема: Чат-бот не відкривається

**Причина:** Неправильний клас у CSS

**Рішення:** Перевірте, що у `style.css` є клас `.active`:
```css
.chatbot-window.active {
  opacity: 1;
  pointer-events: all;
  transform: translateY(0);
}
```

### Проблема: API повертає 404

**Причина:** Неправильна назва моделі

**Рішення:** Оновіть модель у `chatbot.js`:
```javascript
const MW_GEMINI_MODEL = "gemini-2.0-flash-exp"; // Замість застарілої версії
```

### Проблема: Списки в блозі прилипають до краю

**Причина:** Відсутній `padding-left`

**Рішення:** Додайте стилі:
```css
.blog-article ul, 
.blog-article ol {
  padding-left: 25px !important;
  margin-bottom: 20px;
}
```

1. **Fork** репозиторій
2. Створіть **feature branch**: `git checkout -b feature/amazing-feature`
3. **Commit** ваші зміни: `git commit -m 'Add amazing feature'`
4. **Push** до гілки: `git push origin feature/amazing-feature`
5. Відкрийте **Pull Request**

### Гайдлайни:

- Пишіть зрозумілі commit messages
- Додайте коментарі до складного коду
- Оновіть README.md при необхідності
- Тестуйте на різних браузерах
